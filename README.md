## 🎯 게임 목표

캐릭터는 **자동으로 점프**합니다.  
플레이어는 좌우 이동만 조작하여 발판을 밟으며 **최대한 높이** 올라가세요.  
바닥으로 떨어지면 게임 오버!

---

## 🕹️ 조작 방법

| 입력 | 동작 |
|------|------|
| `A` / `←` (왼쪽 방향키) | 왼쪽 이동 |
| `D` / `→` (오른쪽 방향키) | 오른쪽 이동 |
| `Space` / `Enter` | 게임 시작 / 재시작 |
| `R` | 게임 오버 시 즉시 재시작 |
| 모바일 — 화면 **왼쪽** 터치 | 왼쪽 이동 |
| 모바일 — 화면 **오른쪽** 터치 | 오른쪽 이동 |

> 메인 메뉴에서 `A/D` 또는 방향키로 가이드 페이지를 넘길 수 있습니다.

## 🧩 게임 메커니즘

### 캐릭터 (Player)
- 크기: 35 × 35 픽셀, 빨간 둥근 사각형 형태
- 눈이 이동 방향을 따라 움직이는 시각 효과 포함
- **중력** 적용: 매 프레임 `vy += 0.16` 가속
- 이동 속도: 좌우 `4.5px/frame`
- 점프력: `vy = -7.2` (발판 착지 시 자동 점프)

### 화면 가장자리 관통 (Wrap-around)
- 화면 오른쪽 끝을 벗어나면 **왼쪽 끝**에서 등장
- 화면 왼쪽 끝을 벗어나면 **오른쪽 끝**에서 등장
- 전략적인 경로 선택에 활용 가능

### 스크롤 & 점수
- 캐릭터가 화면 상단 40% 지점을 넘으면 **화면 전체가 아래로 스크롤**
- 스크롤된 픽셀만큼 **점수에 누적** (정수 단위)
- 점수는 화면 좌상단에 실시간 표시

## 🟩 발판 (Platform) 종류

| 색상 | 타입 | 설명 |
|------|------|------|
| 🟫 회색 (`#747d8c`) | `base` | 시작 바닥 발판, 화면 전체 너비 |
| 🟢 초록 (`#2ed573`) | `normal` | 일반 발판, 착지 시 정상 점프 |
| 🔵 파랑 (`#1e90ff`) | `moving` | 좌우로 움직이는 이동 발판 |
| 🟠 주황 (`#ffa502`) | `fragile` | 부서지는 발판, 점프력 50% 감소 후 파괴 |

### 발판 생성 규칙
- 시작 시 바닥에서 위 방향으로 110~160px 간격으로 자동 생성
- 발판이 화면 아래로 사라지면 **상단에 새 발판 자동 생성**
- 새로 생성되는 발판은 20% 확률로 `moving`, 80% 확률로 `normal`

## 💀 게임 오버 & 부활

### 게임 오버 조건
- 캐릭터가 화면 하단(`y > CANVAS_HEIGHT`) 아래로 떨어지면 **즉시 게임 오버**

### 게임 오버 화면 구성
- 최종 점수 표시
- **광고 보고 이어하기** 버튼 (부활 기능)
- **처음부터 다시하기** 버튼
- **메인으로 돌아가기** 버튼

### 광고 시청 후 부활 (Ad Revive)
- 광고 시청 버튼 클릭 → 2초간 로딩 바 애니메이션 재생
- 로딩 완료 후 캐릭터가 **화면 중앙에서 부활**, 즉시 점프
- 기존 점수와 발판 상태 유지
- 부활은 **1회** 사용 가능 (부활 후 광고 버튼 숨김)

## 🖥️ UI 구성

```
┌─────────────────────────────┐
│  Score: 0                   │  ← 실시간 점수
│                             │
│        [메인 메뉴]          │
│   ┌─────────────────────┐   │
│   │  JUMPER             │   │  ← 타이틀
│   │  [가이드 슬라이드]  │   │  ← 3페이지 (조작법/목표/교차이동)
│   └─────────────────────┘   │
│                             │
│      [ START GAME ]         │  ← 시작 버튼
│   Space 또는 Enter로 시작   │
└─────────────────────────────┘
```

### 카운트다운 시작
- START GAME 클릭 시 `3 → 2 → 1 → GO!` 카운트다운 (0.7초 간격)
- 카운트다운 종료 후 게임 시작

## 🎨 디자인 특징

- **배경**: 하늘색 그리드 패턴 (`#c4def6` + 반투명 흰 라인)
- **캐릭터**: 빨간 둥근 사각형 + 흰/검은 눈동자 (이동 방향 연동)
- **오버레이**: 다크 반투명 블러 UI (`backdrop-filter: blur`)
- **폰트**: `Inter`, `Segoe UI` 계열
- **버튼**: 초록(`#2ed573`) 기본, 금색(`#f1c40f`) 광고, 회색 보조

## 🔧 주요 함수 설명

| 함수 | 역할 |
|------|------|
| `generatePlatforms()` | 게임 시작 시 발판 초기 배치 |
| `requestStart()` | 카운트다운 후 게임 시작 |
| `startGame()` | 점수/플레이어 초기화 및 루프 시작 |
| `update()` | 매 프레임: 입력 처리, 물리 연산, 충돌 감지 |
| `draw()` | 매 프레임: 발판 및 캐릭터 렌더링 |
| `loop()` | `update() + draw()` 반복 호출 |
| `endGame()` | 게임 오버 처리 및 UI 전환 |
| `watchAd()` | 광고 시뮬레이션 및 부활 연결 |
| `revivePlayer()` | 부활: 캐릭터 위치 복원 및 게임 재개 |
| `changePage(dir)` | 메인 메뉴 가이드 페이지 전환 |
| `resizeCanvas()` | 반응형 캔버스 크기 재조정 |

## 📝 개발 참고사항

- 발판 간 세로 간격은 `110 ~ 160px`으로 설정되어 있으며, 점프력(`-7.2`)과 중력(`0.16`)에 맞게 튜닝되어 있습니다.
- `fragile` 타입 발판은 현재 초기 생성 시에는 포함되지 않으며, 코드에서 타입을 추가하면 도입 가능합니다.
- 광고 시청 기능은 **시뮬레이션**으로 실제 광고 SDK와 연동되어 있지 않습니다.
